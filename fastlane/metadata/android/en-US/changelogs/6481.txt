• Closing a fullscreen desktop tab no longer locks you out of the app. Tapping X while a VNC/X11 tab was fullscreen closed the tab but never left fullscreen: no top bar, no bottom bar, and swiping between tabs is switched off by that same state — so the way out was disabled by the thing you needed to escape. Only force-stopping recovered. Fixed, and a fullscreen state can no longer outlive its session.

• Automations follow a device that changes address again. Haven has been able to re-find a saved host by its SSH key since v5.68.44, but it never actually ran: the check deciding whether the host failed to answer (as opposed to answering and rejecting you) didn't recognise the most common way a moved device fails, because Android words that error differently from the ones it looked for. Tapping the profile by hand had the same blind spot.

• The keyboard can finally read the terminal's input line. Haven never answered when a keyboard asked what was in the field. Most cope, but a predictive keyboard that keeps its own copy of what you are typing has nothing to correct it against, so after Enter it can keep building on the line it already ran — typing ls twice sent lsls. If you use SwiftKey in Standard keyboard mode, try this one.
